仮装環境は多分デバッグ構成見ればわかる
かならずsatcompをカレントディレクトリにして
$ pipenv run python src/cs_solver.py
のように実行する
json2img.pyとtest.pyがバグる

miniforgeで仮想環境一覧を確認
$ conda info --envs
python3.8の環境に入る
$ conda activate [仮想環境名]
作成
$ conda create -n [仮想環境名] python=3.8
ちなみに削除は
$ conda remove -n [仮想環境名] --all

1. condaでpython3.8の環境を作成
$ conda create -n satcomp python=3.8
2. 1の環境に入る
$ conda activate satcomp
3. pipenvをインストールする
$ pip install pipenv
4. パッケージをまとめてインストールする
$ pipenv sync
5. graphvizもインストールする
$ brew install graphviz
6-1. 実行
$ pipenv run python src/cs_solver.py --str abracadabra
6-2. 入力した文字列に対してcs_solver.py, rlslp_solver.py, slp_solver.pyを順に実行し，それぞれjsonではなく画像として出力
$ pipenv run python migita/test.py abracadabra
6-3. ランダムに生成された文字列に対して三つのソルバーを実行する，これを繰り返す
$ pipenv run python migita/test.py

上の手順で作られるファイル構造
conda(satcomp){ (conda listで確認可能)
    graphviz
    pip(python3.8){ (pip listで確認可能)
        pipenv{ (pipenv graphで確認可能)
            pipfileに記述されているパッケージ
        }
    }
}
graphvizをpipでcondaやpipenv内にインストールしてもdotコマンドが認識されない，なぜ？
    pythonパッケージのgraphvizにターミナルで実行できるコマンドは含まれないから(dot -Vで確認できる)
    pipやpipenvではなく，brewやcondaにインストールして解決できる
    pythonの実行ファイルは，conda環境にも，pipenv環境にも存在している．デバッグなどで用いる時は，pipenv環境の中にある実行ファイルを指定する．
